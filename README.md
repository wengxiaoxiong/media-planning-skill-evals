# Media Planning Skill - Evals 出题方案

基于这个 Media Planning skill 的核心能力和约束条件,我设计了3道递进式测试题,覆盖不同难度和场景:

---

## 📋 Test Case 1: 标准场景 - 新品牌冷启动
**难度**: ⭐⭐⭐ (中等)

### 输入 (User Input)
```
我们是一个新锐咖啡品牌"野生咖啡",主打精品手冲咖啡豆,目标人群是25-35岁一二线城市的咖啡爱好者。

产品特点:
- 云南/埃塞俄比亚单一产地豆
- 客单价 120-180 元/包
- 强调风味教育与冲煮仪式感

目标:
- Q2 新品上市,3个月内建立品牌认知
- 预算 80 万
- 期望获得 5000+ 首单用户

请帮我做媒介规划。
```

### 评估维度
| 维度 | 检查点 | 权重 |
|------|--------|------|
| **流程合规性** | ✓ 先生成 NOTES.MD,再生成 report.html<br>✓ NOTES 包含 Evidence Ledger<br>✓ 报告中无 NOTES 外新事实 | 25% |
| **调研质量** | ✓ 搜索咖啡行业媒介数据<br>✓ 查询小红书/抖音咖啡类目数据<br>✓ 竞品分析(三顿半/永璞等) | 20% |
| **推理严谨性** | ✓ 预算分配有计算依据<br>✓ CPA/ROI 估算标注"估算"<br>✓ 渠道选择有人群触媒证据支撑 | 25% |
| **报告质量** | ✓ 至少5类图表且可交互<br>✓ 商务风设计(无emoji/彩色文字)<br>✓ 结论级引用 [S1][S2] | 20% |
| **决策价值** | ✓ 明确渠道优先级与理由<br>✓ 风险提示(如冷启动转化率不确定性)<br>✓ Next Actions 可执行 | 10% |

## 📋 Test Case 2: 高难场景 - 数据稀缺 + 多目标冲突
**难度**: ⭐⭐⭐⭐ (困难)

### 输入 (User Input)
```
我们是一家 B2B SaaS 公司,做"智能合同管理系统",目标客户是中大型企业法务部门。

挑战:
- 这是一个极度垂直的小众市场,公开数据很少
- 需要同时实现品牌教育(70%决策者不知道这个品类存在)和线索获取
- 预算只有 30 万,但销售周期长达 3-6 个月
- 客单价 8-20 万/年

目标:
- 6个月内获得 50 个 MQL(Marketing Qualified Leads)
- 建立"行业专家"认知

请给出媒介策略。
```

### 评估维度
| 维度 | 检查点 | 权重 |
|------|--------|------|
| **数据缺口处理** | ✓ NOTES 中明确列出"无公开数据"项<br>✓ 使用类比推理(如参考其他 B2B SaaS 数据)<br>✓ 标注所有假设条件 | 30% |
| **多目标协调** | ✓ 分析品牌教育与线索获取的资源冲突<br>✓ 给出阶段性投放建议(如前3月品牌/后3月效果)<br>✓ 渠道组合反映长决策周期特性 | 25% |
| **垂直场景适配** | ✓ 不推荐大众平台(抖音/小红书)<br>✓ 考虑专业渠道(LinkedIn/知乎/行业媒体)<br>✓ 内容形式匹配决策者画像 | 20% |
| **诚实度** | ✓ 明确指出"30万预算 vs 50 MQL"可能不匹配<br>✓ 给出"乐观/基准/保守"三种场景<br>✓ 不虚构不存在的行业数据 | 15% |
| **商务规范** | ✓ 报告符合所有设计规范<br>✓ Source Index 完整 | 10% |

## 📋 Test Case 3: 极限场景 - 反向验证与自我约束
**难度**: ⭐⭐⭐⭐⭐ (极难)

### 输入 (User Input)
```
我是某知名饮料品牌市场总监,老板给了我 2000 万预算做 Q4 大促。

我看了很多 Agency 的方案,都说"抖音投 800 万能带来 5000 万 GMV",但我怎么判断这些数字是不是瞎编的?

另外,去年我们在小红书投了 300 万,Agency 说"种草了 1.2 亿人次",但实际销量增长才 8%,我觉得被忽悠了。

你能帮我:
1. 做一份"可审计"的媒介规划吗?(要能让 CFO 看懂每一笔钱怎么花、怎么算回报)
2. 告诉我哪些数字是真的、哪些是"包装过的"?
```

### 评估维度
| 维度 | 检查点 | 权重 |
|------|--------|------|
| **反忽悠能力** | ✓ NOTES 中分析"5000万GMV"声称的合理性<br>✓ 拆解"种草1.2亿人次"的水分来源<br>✓ 给出行业真实 ROI 基准范围 | 30% |
| **可审计性设计** | ✓ 每个渠道预算有明确计算公式<br>✓ 效果预估分"保证数"与"超额数"<br>✓ 提供验证清单(如监测埋点要求) | 25% |
| **诚实与边界** | ✓ 明确指出"某些效果无法精确归因"<br>✓ 区分"平台数据"vs"第三方监测"<br>✓ 不承诺不可能达成的 ROI | 20% |
| **CFO 视角** | ✓ 报告包含财务敏感性分析<br>✓ 现金流与回款周期考量<br>✓ 可量化的风险对冲方案 | 15% |
| **专业对抗性** | ✓ 能指出常见 Agency 话术陷阱<br>✓ 给出"二次验证"方法论<br>✓ 提供同行 benchmark 对比 | 10% |

---

## 🎯 评分标准汇总

### 通过标准
- **Test 1**: ≥ 75 分(基础能力合格)
- **Test 2**: ≥ 65 分(复杂场景可用)
- **Test 3**: ≥ 60 分(专业级防御能力)

### 关键失败模式
1. **流程违规**: 未生成 NOTES 或报告中出现 NOTES 外新事实 → 直接 Fail
2. **数据造假**: 虚构不存在的行业数据且未标注"估算" → 直接 Fail
3. **忽视约束**: 使用 emoji、彩色文字、非 ECharts 图表 → 扣 20 分
4. **无 Source**: 关键结论无 [S1] 引用 → 每处扣 5 分
